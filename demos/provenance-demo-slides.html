01 · The problem

What you can’t see in docker pull

A registry tag is a mutable label that anyone with push rights can move. A pulled binary is just bytes. Without provenance you can’t answer the three questions a serious operator has to ask.

QuestionWhat proves it
Did NVIDIA actually build this?Sigstore signature pinned to NVIDIA’s GitHub Actions identity
From which source, on which workflow?SLSA Build Provenance v1: repo + commit SHA + workflow ref
What’s inside it?SPDX SBOM attestation: every package, library, license

Tags lie

ghcr.io/nvidia/aicr:v1.2.3 is mutable. A repush replaces it. Tag-only pulls have zero integrity.

Digests don’t lie

@sha256:f0c1… is content-addressed. Resolve once, then anchor every check to it.

03 · Sign

cosign → Rekor, the keyless way

Why keyless No long-lived signing key to leak. The signer’s identity is the workflow ref (https://github.com/NVIDIA/aicr/.github/workflows/on-tag.yaml@refs/tags/vX.Y.Z) — verifiable against Fulcio’s cert chain and Rekor’s transparency log without contacting NVIDIA.
04 · Verify · images

Pin the digest, verify the chain

resolve + verify
# 1. Resolve the mutable tag to the immutable digest, once.
$ TAG=$(curl -s https://api.github.com/repos/NVIDIA/aicr/releases/latest | jq -r .tag_name)
$ IMAGE=ghcr.io/nvidia/aicr
$ DIGEST=$(crane digest "$IMAGE:$TAG")
$ echo "$IMAGE@$DIGEST"
ghcr.io/nvidia/aicr@sha256:f0c1abc…

# 2. Verify the SLSA provenance attestation against NVIDIA's identity.
$ gh attestation verify oci://"$IMAGE@$DIGEST" --repo NVIDIA/aicr --signer-workflow NVIDIA/aicr/.github/workflows/on-tag.yaml --source-ref "refs/tags/${TAG}"
Loaded digest sha256:f0c1abc… for oci://ghcr.io/nvidia/aicr@…
✓ Verification succeeded!

The following policy criteria will be enforced:
  - OIDC Issuer must match: https://token.actions.githubusercontent.com
  - Source Repository Owner URI must match: https://github.com/NVIDIA
  - Predicate type must match: https://slsa.dev/provenance/v1

Everything that follows anchors to $IMAGE@$DIGEST. Tags only label what we already pinned.

04 · Verify · SBOM

SPDX SBOM, signed and queryable

cosign verify-attestation + grype
# Pull and verify the SPDX SBOM attestation.
$ cosign verify-attestation --type spdxjson \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com \
    --certificate-identity-regexp '^https://github\.com/NVIDIA/aicr/\.github/workflows/on-tag\.yaml@refs/tags/.+$' \
    "$IMAGE@$DIGEST" > predicate.json

# Vulnerability scan, straight off the SBOM (no re-pull).
$ jq -r .payload predicate.json | base64 -d | jq .predicate > sbom.json
$ grype sbom:./sbom.json
NAME            INSTALLED   FIXED-IN   TYPE        VULNERABILITY   SEVERITY
stdlib          1.26.0      —          go-module   GHSA-…          Low

# License compliance, one jq away.
$ jq -r '.packages[]
        | select(.licenseDeclared != "NOASSERTION")
        | "\(.name) \(.versionInfo) \(.licenseDeclared)"' sbom.json | head -3
github.com/spf13/cobra v1.10.1 Apache-2.0
k8s.io/client-go      v0.34.2 Apache-2.0
sigs.k8s.io/yaml      v1.6.0  MIT
The SBOM is the source of truth Vulnerability scanners and license tools read SPDX directly — no need to re-pull, re-scan, or re-derive. The signature pins the SBOM to the exact image digest.

06 · Enforce

Block unsigned pulls at admission

Verification at docker pull is opt-in. To make it a property of the cluster, enforce it with an admission controller — either Sigstore Policy Controller (validated for AICR) or Kyverno (its bundle support was cluster-tested and does not yet verify AICR — see #1537).

ClusterImagePolicy (sigstore)
apiVersion: policy.sigstore.dev/v1beta1
kind: ClusterImagePolicy
metadata: { name: aicr-images-require-attestation }
spec:
  images:
  - glob: "ghcr.io/nvidia/aicr**"
  authorities:
  - signatureFormat: bundle
    keyless:
      url: https://fulcio.sigstore.dev
      identities:
      - issuer: https://token.actions.githubusercontent.com
        subjectRegExp: '^https://github\.com/NVIDIA/aicr/\.github/workflows/on-tag\.yaml@refs/tags/.+$'
    ctlog: { url: https://rekor.sigstore.dev }
    attestations:
    - name: build-provenance
      predicateType: https://slsa.dev/provenance/v1

Allowed

kubectl run a --image=ghcr.io/nvidia/aicr:vX.Y.Z — has attestation, identity matches.

Rejected

A ghcr.io/nvidia/aicr* image without a valid release attestation — no matching attestations found. Images outside the glob (e.g. nginx:latest) are not evaluated.

07 · Where it fits

One trust model, three subjects

Images & binaries

SLSA provenance + signed SPDX SBOM on every published image (CLI binaries ship a separate SBOM asset), signed by NVIDIA CI. You are here.

Deployment bundles

aicr bundle --attest signs the generated bundle with build provenance + trust levels. demos/bundle-attestation.md

Recipe evidence

A signer-bound validation result for a recipe (proves who signed it, not cluster physicality), OCI-distributed with an in-tree pointer. demos/evidence.md

Same primitives

Sigstore keyless + Fulcio + Rekor + in-toto across the board — different subjects, one verification model.

Summary Tag → digest → gh attestation verify → cosign verify-attestation for the SBOM → Rekor lookup → admission policy. Five commands, zero trust in the registry.
